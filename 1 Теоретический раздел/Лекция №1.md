# Введение в Инструментальное программное обеспечение (Python)

## Цели и задачи лекции
- Понять роль и назначение инструментального программного обеспечения в современном программировании.
- Ознакомиться с особенностями языка Python как инструмента для разработки.
- Научиться использовать основные средства разработки: IDE, менеджеры пакетов, виртуальные окружения.
- Освоить базовые практики работы с кодом: написание, отладка, тестирование.

## Ключевые понятия
1. **Инструментальное ПО** – программные средства, облегчающие разработку, тестирование и развертывание приложений.  
2. **IDE** (Integrated Development Environment) – интегрированная среда разработки, объединяющая редактор кода, отладчик, менеджер зависимостей и другие инструменты.  
3. **Виртуальное окружение** – изолированное пространство, в котором устанавливаются пакеты, не влияя на системные библиотеки.  
4. **Менеджер пакетов** – утилита, упрощающая установку, обновление и удаление библиотек (например, pip).  
5. **Отладчик** – инструмент, позволяющий пошагово выполнять программу, просматривать переменные и трассировать ошибки.  
6. **Тестирование** – процесс проверки корректности работы кода с помощью автоматизированных тестов.  
7. **Скрипт** – короткая программа, выполняемая напрямую интерпретатором.  
8. **Модуль** – файл с расширением `.py`, содержащий функции, классы и переменные, которые можно импортировать.  
9. **Пакет** – каталог с модулями и файлом `__init__.py`, позволяющий группировать связанные модули.  
10. **CLI** (Command Line Interface) – интерфейс взаимодействия с программой через командную строку.

### 1.4 Управление зависимостями (6 мин)
- Что такое `requirements.txt` и `pip`.
- Создание и активирование виртуального окружения (`venv`).
- Пример:  
  ```bash
  python -m venv env
  source env/bin/activate  # Linux/Mac
  env\Scripts\activate     # Windows
  pip install requests
  ```

### 1.5 Отладка и тестирование (8 мин)
- Использование встроенного отладчика IDE.
- Написание простых unit-тестов с `unittest`.  
  ```python
  import unittest
  from math import sqrt

  class TestMath(unittest.TestCase):
      def test_sqrt(self):
          self.assertEqual(sqrt(4), 2)

  if __name__ == "__main__":
      unittest.main()
  ```

### 1.6 Практический пример: простое скриптовое приложение (6 мин)
- Задача: скрипт, читающий CSV, выводящий статистику.  
  ```python
  import csv
  from statistics import mean

  def read_csv(path):
      with open(path, newline='') as f:
          return list(csv.DictReader(f))

  def avg_age(data):
      ages = [int(row['age']) for row in data]
      return mean(ages)

  if __name__ == "__main__":
      data = read_csv('people.csv')
      print(f"Average age: {avg_age(data):.2f}")
  ```
