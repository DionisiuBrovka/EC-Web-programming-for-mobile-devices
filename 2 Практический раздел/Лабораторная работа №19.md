# Лабораторная работа № 19. Обработка данных в формате JSON на стороне клиента

## Цель и задачи

1. Познакомиться с форматом JSON как с текстовым представлением структурированных данных.  
2. Научиться сериализовать Python‑объекты в JSON и десериализовать JSON в объекты Python.  
3. Изучить методы работы с данными в формате JSON на стороне клиента (JavaScript) через fetch API.  
4. Реализовать простое клиентское приложение, которое загружает данные из внешнего API в формате JSON, обрабатывает их и отображает на веб‑странице.

## Теоретическая часть

| Понятие | Определение |
|---------|-------------|
| **JSON** | Язык сериализации данных, основанный на синтаксисе JavaScript, использующий пары «ключ‑значение» и массивы. |
| **Сериализация** | Процесс преобразования объекта в последовательность байт (или строку) для хранения или передачи. |
| **Десериализация** | Обратный процесс – преобразование последовательности байт (или строки) в объект. |
| **`json` модуль** | Стандартный Python‑модуль для работы с JSON‑данными (`json.dumps`, `json.loads`). |
| **`fetch`** | Встроенный в браузер API для выполнения HTTP‑запросов. |
| **Promise** | Объект, представляющий результат асинхронной операции, который может быть «разрешён» или «отклонён». |
| **CORS** | Политика безопасности браузера, ограничивающая доступ к ресурсам, находящимся на другом домене. |
| **`Content-Type: application/json`** | Заголовок HTTP, указывающий, что тело запроса/ответа содержит JSON‑данные. |
| **`JSON.parse`** | Встроенная JavaScript‑функция, преобразующая строку JSON в объект. |
| **`JSON.stringify`** | Встроенная JavaScript‑функция, преобразующая объект в строку JSON. |

## Ход выполнения работы

1. **Подготовка окружения**  
   - Создайте папку проекта `json_lab19`.  
   - Внутри создайте файлы `index.html`, `app.js`, `style.css`.  

2. **Создание HTML‑страницы**  
   ```html
   <!DOCTYPE html>
   <html lang="ru">
   <head>
       <meta charset="UTF-8">
       <title>JSON Lab 19</title>
       <link rel="stylesheet" href="style.css">
   </head>
   <body>
       <h1>Список пользователей</h1>
       <button id="loadBtn">Загрузить данные</button>
       <ul id="userList"></ul>

       <script src="app.js"></script>
   </body>
   </html>
   ```  

3. **Запрос данных из внешнего API**  
   В `app.js` реализуем функцию `loadUsers()`, которая делает GET‑запрос к `https://jsonplaceholder.typicode.com/users`.  
   ```js
   const loadBtn = document.getElementById('loadBtn');
   const userList = document.getElementById('userList');

   async function loadUsers() {
       try {
           const response = await fetch('https://jsonplaceholder.typicode.com/users');
           if (!response.ok) throw new Error('Network response was not ok');
           const users = await response.json(); // десериализация
           displayUsers(users);
       } catch (error) {
           console.error('Ошибка загрузки:', error);
       }
   }

   function displayUsers(users) {
       userList.innerHTML = '';
       users.forEach(u => {
           const li = document.createElement('li');
           li.textContent = `${u.name} (${u.email})`;
           userList.appendChild(li);
       });
   }

   loadBtn.addEventListener('click', loadUsers);
   ```

4. **Обработка ошибок и CORS**  
   - Проверяем `response.ok`.  
   - В случае ошибки выводим сообщение в консоль.  
   - Поскольку API поддерживает CORS, запрос проходит без дополнительных настроек.

5. **Сериализация данных в Python (опционально)**  
   Для демонстрации сериализации создаём скрипт `save_users.py`.  
   ```python
   import json
   import requests

   resp = requests.get('https://jsonplaceholder.typicode.com/users')
   users = resp.json()  # десериализация

   with open('users.json', 'w', encoding='utf-8') as f:
       json.dump(users, f, ensure_ascii=False, indent=4)  # сериализация
   ```

6. **Проверка результата**  
   - Откройте `index.html` в браузере.  
   - Нажмите «Загрузить данные» и убедитесь, что список пользователей появился.  
   - Откройте консоль браузера и проверьте отсутствие ошибок.  

## Контрольные вопросы / задания для самопроверки

1. Что такое JSON и какие типы данных поддерживает этот формат?  
2. В чем разница между `json.dumps` и `json.loads`?  
3. Какой заголовок необходимо установить в запросе, если тело содержит JSON‑данные?  
4. Что произойдёт, если в `fetch` вызвать `response.json()` и тело ответа не будет валидным JSON?  
5. Как в JavaScript преобразовать объект в строку JSON?  
6. Почему важно проверять `response.ok` при работе с `fetch`?  
7. Что такое CORS и как он влияет на запросы к сторонним API?  
8. Напишите пример кода, который отправляет POST‑запрос с JSON‑тело в Python.  
9. Как сериализовать список словарей в JSON с отступами 2 пробела?  
10. Как вывести в консоль сообщение, если запрос к API завершился ошибкой 404?
