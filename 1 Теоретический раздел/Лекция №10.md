# Определение классов в JavaScript

## Цели и задачи лекции
- Понять, как в JavaScript реализуются объекты‑ориентированные конструкции через классы.
- Научиться объявлять классы, наследовать их и использовать методы и свойства.
- Ознакомиться с особенностями синтаксиса ES6 и более поздних версий.
- Практически применить знания, создавая простые программы на JavaScript.

## Ключевые понятия
- **Класс** – шаблон для создания объектов с определённой структурой и поведением.  
- **Конструктор** – специальный метод `constructor`, вызываемый при создании экземпляра.  
- **Метод** – функция, объявленная внутри класса.  
- **Наследование** – механизм, позволяющий создавать новый класс на основе существующего.  
- **Полиморфизм** – способность объектов разных классов обрабатывать одинаковые вызовы методов по‑разному.  
- **Статический метод** – метод, привязанный к классу, а не к его экземплярам.  
- **Модуль** – отдельный файл, экспортирующий и импортирующий классы.  
- **Инкапсуляция** – скрытие внутренней реализации объекта от внешнего кода.  
- **Тестирование** – проверка корректности работы классов при помощи unit‑тестов.  
- **Интерфейс** – набор методов, которые должен реализовать класс (в JavaScript — соглашение).

## Основное содержание

### 1. Введение в синтаксис классов (10 минут)
- Краткая история: от прототипного наследования к синтаксису ES6.  
- Пример простого класса `Person`:
  ```js
  class Person {
    constructor(name, age) {
      this.name = name;
      this.age = age;
    }
    greet() {
      console.log(`Привет, меня зовут ${this.name}!`);
    }
  }
  ```

### 2. Создание и работа с экземплярами (15 минут)
- Создание объектов: `const alice = new Person('Alice', 30);`.  
- Доступ к свойствам и методам.  
- Обзор `this` в контексте классов.

### 3. Наследование и суперклассы (20 минут)
- Синтаксис `extends` и вызов `super()`:
  ```js
  class Employee extends Person {
    constructor(name, age, salary) {
      super(name, age);
      this.salary = salary;
    }
    work() {
      console.log(`${this.name} работает за ${this.salary}₽`);
    }
  }
  ```
- Переопределение методов и использование `super.methodName()`.

### 4. Статические методы и свойства (10 минут)
- Объявление: `static info() { ... }`.  
- Использование: `Employee.info();`.  
- Пример: `class MathUtils { static add(a, b) { return a + b; } }`.

### 5. Инкапсуляция с помощью символов и приватных полей (10 минут)
- Символьные свойства: `const #privateField = Symbol('private');`.  
- Приватные поля: `#privateField = 0;` (ES2022+).  
- Ограничения доступа из вне.

### 6. Работа с модулями (5 минут)
- Экспорт/импорт классов: `export default class ...` и `import MyClass from './MyClass.js';`.  
- Практика: разделение кода на несколько файлов.

## Примеры и иллюстрации

### Полный пример: иерархия классов
```js
// person.js
export default class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  greet() {
    console.log(`Привет, меня зовут ${this.name}!`);
  }
}

// employee.js
import Person from './person.js';
export default class Employee extends Person {
  constructor(name, age, position) {
    super(name, age);
    this.position = position;
  }
  work() {
    console.log(`${this.name} работает в роли ${this.position}.`);
  }
}

// main.js
import Employee from './employee.js';
const dev = new Employee('Ivan', 28, 'Developer');
dev.greet(); // Привет, меня зовут Ivan!
dev.work(); // Ivan работает в роли Developer.
```

### Пример использования статических методов
```js
class Counter {
  static count = 0;
  constructor() {
    Counter.count++;
  }
  static getCount() {
    return Counter.count;
  }
}
new Counter();
new Counter();
console.log(Counter.getCount()); // 2
```

## Выводы по лекции
- Классы в JavaScript предоставляют удобный способ организации объектно‑ориентированного кода.  
- Синтаксис ES6 упрощает объявление конструкторов, методов, наследования и статических членов.  
- Правильное использование инкапсуляции повышает читаемость и защищённость кода.  
- Модули позволяют структурировать проект, разделяя логику на независимые части.
