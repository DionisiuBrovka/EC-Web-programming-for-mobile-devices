# Лабораторная работа № 17. Назначение обработчиков событий в сценариях на языке JavaScript. Использование разных моделей событий

## Цель и задачи
1. Познакомить студентов с принципами обработки событий в JavaScript.  
2. Научить назначать обработчики событий как через атрибуты DOM, так и через методы `addEventListener` и `removeEventListener`.  
3. Исследовать влияние разных моделей событий (протоколирование, делегирование, bubble vs capture).  
4. Разработать простое интерактивное приложение, демонстрирующее работу выбранных моделей.

## Теоретическая часть
1. **Событие** – асинхронное событие, генерируемое элементом DOM (например, `click`, `keydown`).  
2. **Обработчик события** – функция, привязанная к событию, которая исполняется при его возникновении.  
3. **EventTarget** – объект, к которому можно привязать обработчики (`Element`, `Document`, `Window`).  
4. **addEventListener(type, listener, options)** – метод привязки обработчика.  
5. **removeEventListener(type, listener, options)** – метод удаления обработчика.  
6. **Event propagation** – два фазы: *capturing* (сверху вниз) и *bubbling* (снизу вверх).  
7. **Event delegation** – привязка одного обработчика к родительскому элементу, который перехватывает события от потомков.  
8. **Event object** – объект, передаваемый в обработчик, содержит свойства `type`, `target`, `currentTarget`, `preventDefault()`, `stopPropagation()`.  
9. **Synthetic events** – события, созданные программно через `new Event(...)`.  
10. **Passive listeners** – опция `{passive: true}` для оптимизации скроллинга и производительности.

## Ход выполнения работы
1. **Создать HTML‑страницу**  
   ```html
   <!DOCTYPE html>
   <html lang="ru">
   <head>
     <meta charset="UTF-8">
     <title>Lab 17</title>
     <style>
       .box { width: 200px; height: 200px; background: #f0f0f0; margin: 20px; }
       .inner { width: 100px; height: 100px; background: #c0e0ff; margin: 10px; }
     </style>
   </head>
   <body>
     <div id="outer" class="box">
       <div id="inner" class="inner"></div>
     </div>
     <script src="lab17.js"></script>
   </body>
   </html>
   ```
   *Пояснение:* создаём два вложенных блока для демонстрации bubble и capture.

2. **Назначить обработчики через `addEventListener`**  
   ```js
   // lab17.js
   const outer = document.getElementById('outer');
   const inner = document.getElementById('inner');

   outer.addEventListener('click', e => {
     console.log('Outer (bubble)');
   });

   inner.addEventListener('click', e => {
     console.log('Inner (bubble)');
   });
   ```
   *Пояснение:* два обработчика, оба в фазе bubble (по умолчанию).

3. **Переписать обработчики на фазу capture**  
   ```js
   outer.addEventListener('click', e => {
     console.log('Outer (capture)');
   }, {capture: true});

   inner.addEventListener('click', e => {
     console.log('Inner (capture)');
   }, {capture: true});
   ```
   *Пояснение:* флаг `{capture: true}` переключает фазу.

4. **Реализовать делегирование**  
   ```js
   document.addEventListener('click', e => {
     if (e.target.matches('.inner')) {
       console.log('Delegated: inner clicked');
     }
   });
   ```
   *Пояснение:* один обработчик на `document`, проверяем `e.target`.

5. **Удалить обработчик**  
   ```js
   function removeInner() {
     inner.removeEventListener('click', innerClickHandler);
   }
   ```
   *Пояснение:* демонстрация удаления.

6. **Сгенерировать синтетическое событие**  
   ```js
   const clickEvent = new Event('click', {bubbles: true});
   inner.dispatchEvent(clickEvent);
   ```
   *Пояснение:* создаём событие программно и диспатчим.

7. **Проверка passive listeners**  
   ```js
   window.addEventListener('scroll', () => console.log('Scroll!'), {passive: true});
   ```
   *Пояснение:* оптимизация скроллинга.

8. **Собрать результаты**  
   Откройте консоль браузера и кликните по элементам, наблюдайте порядок вывода.

## Контрольные вопросы / задания для самопроверки
1. Что такое `EventTarget` и какие объекты в DOM являются его реализациями?  
2. Какой порядок выполнения обработчиков, если для одного события заданы несколько `addEventListener`?  
3. Что произойдет, если в обработчике вызвать `e.stopPropagation()`?  
4. В чем отличие синтетического события от естественного?  
5. Как реализовать делегирование событий и какие преимущества оно предоставляет?  
6. Что делает опция `{passive: true}` и в каких случаях её использовать?  
7. Как удалить обработчик, который был назначен через `addEventListener`?  
8. Какой фаза событий будет активна, если не указать параметр `capture`?  
9. Как программно сгенерировать событие `keyup` со значением `Enter`?  
10. Какие свойства объекта `Event` наиболее полезны при работе с обработчиками?
