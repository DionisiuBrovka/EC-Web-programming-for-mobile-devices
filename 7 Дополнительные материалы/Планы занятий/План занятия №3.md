# Физическое и логическое форматирование текста в веб-документе. Внедрение контента на веб-страницу

## Цель занятия  
1. Познакомить студентов с принципами физического и логического форматирования текста в HTML и CSS.  
2. Научить применять эти принципы для создания структурированного и адаптивного веб‑контента с использованием Python‑сценариев.

## Задачи занятия  
- Выделить различия между физическим и логическим форматированием.  
- Рассмотреть основные теги и свойства, отвечающие за структуру и стили текста.  
- Ознакомиться с методами генерации HTML‑контента из Python (Jinja2, Flask, Markdown).  
- Практически применить полученные знания, создав простую веб‑страницу с форматированным текстом.  
- Оценить уровень усвоения материала через контрольные вопросы и мини‑задачу.

## План занятия (90 минут)

- Организационный момент — 5 мин  
- Актуализация знаний — 10 мин  
- Изучение нового материала  
  - Физическое форматирование в HTML — 15 мин  
  - Логическое форматирование в CSS — 15 мин  
  - Генерация HTML из Python — 10 мин  
- Закрепление  
  - Практическая работа (создание страницы) — 20 мин  
  - Обсуждение результатов — 5 мин  
- Итог занятия — 5 мин  

## Содержание этапов

- **Организационный момент** – приветствие, проверка присутствия, краткое напоминание о предыдущем занятии.  
- **Актуализация знаний** – быстрый опрос о том, какие теги и свойства уже знакомы, обсуждение примеров форматирования.  
- **Физическое форматирование в HTML** – разбор семантических тегов (`<h1>–<h6>`, `<p>`, `<blockquote>`, `<code>`, `<pre>`), их влияние на структуру документа.  
- **Логическое форматирование в CSS** – свойства шрифтов, цвета, отступов, выравнивания, использование Flexbox для расположения блоков.  
- **Генерация HTML из Python** – ввод в шаблонизаторы (Jinja2), подключение к Flask, преобразование Markdown в HTML через `markdown`‑библиотеку.  
- **Практическая работа** – студенты создают небольшую веб‑страницу с заголовками, списками, цитатами и вставкой кода, генерируя её через Python‑скрипт.  
- **Обсуждение результатов** – обмен кодом, выявление ошибок, коллективное исправление.  
- **Итог занятия** – подведение итогов, ответы на вопросы, раздача чек‑листа ключевых понятий.

## Оборудование и материалы  
- ПК со свободным доступом в интернет.  
- Проектор/экран.  
- IDE (PyCharm, VS Code) или онлайн‑редактор (Replit).  
- Установленные пакеты: `jinja2`, `flask`, `markdown`.  
- Учебник по HTML/CSS и Python.  

## Контроль и оценивание  
- Вопросы устного теста (5–7 вопросов) в конце занятия.  
- Мини‑задача: создание страницы с заданной структурой (проверка кода и визуального результата).  
- Оценка по критериям: корректность разметки, стилизация, работа шаблона, чистота кода.  

## Домашнее задание  
1. Создать веб‑страницу, используя Jinja2‑шаблон, которая выводит список статей из JSON‑файла.  
2. Включить в страницу стилизацию заголовков и блоков кода, используя CSS‑файл.  
3. Сохранить проект в репозиторий GitHub и отправить ссылку на репозиторий преподавателю.
