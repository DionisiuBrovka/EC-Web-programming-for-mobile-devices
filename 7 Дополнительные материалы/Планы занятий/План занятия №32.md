# Лабораторная работа № 18. Проверка корректности информации, введенной пользователем в элементы формы. Обработка данных формы средствами JavaScript

## Цель занятия
1. Научить студентов реализовывать валидацию данных, вводимых в HTML‑форму, при помощи JavaScript.  
2. Развить навыки отладки и тестирования клиентской логики ввода.

## Задачи занятия
- Ознакомить с правилами валидации форм (required, pattern, minlength, maxlength).  
- Практически применить события `input`, `blur` и `submit` для проверки данных.  
- Разработать функцию вывода сообщений об ошибках рядом с полями формы.  
- Проверить корректность работы кода на разных браузерах.  
- Подготовить короткий отчёт о проделанной работе.

## План занятия (90 мин)

- Организационный момент — 5 мин  
- Актуализация знаний — 10 мин  
- Изучение нового материала — 20 мин  
- Практическая часть (рабочий блок) — 35 мин  
- Закрепление — 10 мин  
- Итог занятия — 10 мин  

## Содержание этапов

- **Организационный момент**  
  Проверка присутствия, раздача ноутбуков, запуск IDE, проверка доступа к интернету.

- **Актуализация знаний**  
  Быстрый опрос по предыдущему материалу: какие события JavaScript применяются для обработки форм? Что такое `event.preventDefault()`?  

- **Изучение нового материала**  
  Рассмотрение примеров валидации: required, pattern, minlength, maxlength, custom функции проверки. Обсуждение стратегий вывода сообщений об ошибках.

- **Практическая часть (рабочий блок)**  
  Студенты в парах создают простую форму (имя, email, пароль) и реализуют клиентскую валидацию. Валидация включает:  
  1. Проверку заполнения обязательных полей.  
  2. Проверку формата email через регулярное выражение.  
  3. Проверку длины пароля (не менее 8 символов).  
  4. Вывод сообщений об ошибках рядом с полями.  
  5. Предотвращение отправки при ошибках.  
  Преподаватель проводит мини‑проверку кода, уточняет вопросы.

- **Закрепление**  
  Краткий тест из 5 вопросов (выбор правильного ответа) и обсуждение решений.  

- **Итог занятия**  
  Обсуждение трудностей, выводы по работе, раздача чек‑листа по валидации форм.  

## Оборудование и материалы
- Персональные компьютеры с установленной IDE (VS Code, PyCharm, Sublime Text).  
- Браузер Chrome/Firefox с поддержкой JavaScript.  
- Проектор для демонстрации кода.  
- Учебник: «Инструментальное программное обеспечение (изучение Python)» – глава «Работа с формами в веб‑приложениях».  
- Файл шаблона HTML + JavaScript для лабораторной работы.  

## Контроль и оценивание
- **Формативный контроль**: наблюдение за работой в парах, вопросы во время практики.  
- **Тест**: 5 вопросов с выбором ответа, оценка 0–5 баллов.  
- **Критерии оценки лабораторной работы**:  
  1) Корректная реализация всех требований валидации (30 %).  
  2) Чистота и понятность кода (20 %).  
  3) Отображение сообщений об ошибках (20 %).  
  4) Эффективность использования событий (15 %).  
  5) Тестирование кода (15 %).  

## Домашнее задание
1. Расширить форму, добавив поле «Пароль повторно» и проверку совпадения паролей.  
2. Написать простую серверную валидацию (Python Flask) для тех же полей и сравнить результаты с клиентской валидацией.
