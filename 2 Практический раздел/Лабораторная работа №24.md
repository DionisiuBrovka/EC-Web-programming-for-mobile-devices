# Лабораторная работа № 24  
Получение данных с формы методами GET и POST. Обработка данных формы средствами PHP

## Цель и задачи  
1. Освоить работу с HTTP‑методами GET и POST при передаче данных из веб‑форм.  
2. Научиться получать и обрабатывать данные, отправленные из формы, в PHP‑скриптах.  
3. Понять принципы безопасности при работе с пользовательским вводом (экранирование, валидация).  
4. Разработать простое веб‑приложение, демонстрирующее передачу и вывод введённых данных.

## Теоретическая часть  
| Понятие | Описание |
|---------|----------|
| **HTTP‑метод GET** | Передаёт параметры в URL‑строке. Идёт в виде `?name=value&age=25`. Ограничения: длина строки, видимость данных. |
| **HTTP‑метод POST** | Передаёт данные в теле запроса. Не видны в URL, не ограничены длиной. |
| **`$_GET`** | Ассоциативный массив в PHP, содержащий параметры, переданные через GET. |
| **`$_POST`** | Ассоциативный массив в PHP, содержащий параметры, переданные через POST. |
| **`$_REQUEST`** | Объединяет `$_GET`, `$_POST` и `$_COOKIE`. |
| **`htmlspecialchars()`** | Функция, экранирующая специальные HTML‑символы, защищающая от XSS. |
| **`filter_input()`** | Функция фильтрации и валидации входных данных. |
| **Кросс‑сайтовая подделка запросов (CSRF)** | Угроза, при которой злоумышленник заставляет пользователя отправить запрос с чужим браузером. |
| **Механизм защиты от CSRF** | Использование токенов (hidden‑полей), проверка `Referer` и `Origin`. |
| **`enctype="multipart/form-data"`** | Атрибут формы, необходимый для загрузки файлов. |

## Ход выполнения работы  
1. **Создание структуры проекта**  
   ```text
   lab24/
   ├── index.html
   ├── process_get.php
   └── process_post.php
   ```  
   Пояснение: `index.html` – страница с формой; два скрипта обрабатывают GET и POST.

2. **Разработка формы (index.html)**  
   ```html
   <!DOCTYPE html>
   <html lang="ru">
   <head>
       <meta charset="UTF-8">
       <title>Форма ввода</title>
   </head>
   <body>
       <h1>Заполните форму</h1>
       <form action="process_get.php" method="get">
           <label>Имя: <input type="text" name="name"></label><br>
           <label>Возраст: <input type="number" name="age"></label><br>
           <input type="submit" value="Отправить GET">
       </form>

       <form action="process_post.php" method="post">
           <label>Имя: <input type="text" name="name"></label><br>
           <label>Возраст: <input type="number" name="age"></label><br>
           <input type="submit" value="Отправить POST">
       </form>
   </body>
   </html>
   ```

3. **Обработка GET‑запроса (process_get.php)**  
   ```php
   <?php
   // Получаем данные из массива $_GET
   $name = $_GET['name'] ?? '';
   $age  = $_GET['age']  ?? '';

   // Экранируем вывод
   $name = htmlspecialchars($name, ENT_QUOTES, 'UTF-8');
   $age  = htmlspecialchars($age, ENT_QUOTES, 'UTF-8');

   echo "<h2>Результат GET‑запроса</h2>";
   echo "<p>Имя: {$name}</p>";
   echo "<p>Возраст: {$age}</p>";
   ?>
   ```

4. **Обработка POST‑запроса (process_post.php)**  
   ```php
   <?php
   // Валидация и фильтрация входных данных
   $name = filter_input(INPUT_POST, 'name', FILTER_SANITIZE_STRING) ?? '';
   $age  = filter_input(INPUT_POST, 'age', FILTER_VALIDATE_INT) ?? 0;

   // Проверка корректности возраста
   if ($age <= 0) {
       $age = 'неизвестно';
   }

   // Экранируем вывод
   $name = htmlspecialchars($name, ENT_QUOTES, 'UTF-8');

   echo "<h2>Результат POST‑запроса</h2>";
   echo "<p>Имя: {$name}</p>";
   echo "<p>Возраст: {$age}</p>";
   ?>
   ```

5. **Тестирование**  
   - Открыть `index.html` в браузере.  
   - Заполнить поля и отправить через GET → проверить отображение данных в URL и вывод на `process_get.php`.  
   - Повторить через POST → убедиться, что данные не видны в URL и корректно выводятся на `process_post.php`.  
   - Проверить работу с пустыми полями и некорректными значениями (например, буквы в поле возраста).

6. **Анализ результатов**  
   - Сравнить видимость данных в GET и POST.  
   - Оценить эффективность экранирования и валидации.  
   - Обсудить потенциальные угрозы (XSS, CSRF) и способы их устранения.

## Контрольные вопросы / задания для самопроверки  
1. Как отличить GET‑запрос от POST‑запроса в URL‑строке?  
2. Что хранит массив `$_GET`? `$_POST`? `$_REQUEST`?  
3. Почему при работе с пользовательским вводом необходимо использовать `htmlspecialchars()`?  
4. Как можно защититься от XSS‑атак в выводе пользовательских данных?  
5. Какой механизм позволяет избежать CSRF‑атак при работе с формами?  
6. В чем разница между `filter_input()` и простым чтением из `$_POST`?  
7. Что произойдёт, если в поле возраста пользователь введёт строку «abc» при использовании метода POST?  
8. Как изменить форму так, чтобы отправлять данные методом POST, но все равно отображать их в URL (не рекомендуется)?  
9. Как можно добавить в форму скрытое поле (hidden) для передачи токена CSRF?  
10. Как можно проверить, пришёл ли запрос из того же сайта (проверка `Referer`/`Origin`)?
