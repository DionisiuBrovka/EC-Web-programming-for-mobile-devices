# Архитектура и дизайн сайта. Структура веб‑документа

## Цели и задачи лекции
- Понять основные принципы архитектуры веб‑сайта.
- Овладеть понятиями дизайна и структуры веб‑документа.
- Научиться строить логическую иерархию страниц.
- Применить полученные знания при создании простого прототипа сайта.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| **Архитектура сайта** | Структурная модель, определяющая взаимосвязь страниц и компонентов. |
| **Дизайн интерфейса** | Визуальная иерархия, стилизация и пользовательский опыт. |
| **HTML‑документ** | Текстовый файл с разметкой, определяющий структуру веб‑страницы. |
| **CSS‑стили** | Правила, задающие внешний вид элементов HTML‑документа. |
| **DOM‑дерево** | Объектная модель документа, отражающая структуру страницы. |
| **Модульность** | Разделение кода на независимые, переиспользуемые части. |
| **Responsive Design** | Адаптивность интерфейса под различные экраны. |
| **SEO‑структура** | Организация контента для улучшения индексации поисковыми системами. |
| **Кроссбраузерность** | Совместимость сайта с разными браузерами. |
| **Кастомизация** | Возможность изменить внешний вид без изменения логики. |

## Основное содержание

### 1. Архитектура веб‑сайта (12 мин)
- Иерархия страниц: главная → разделы → подстраницы.
- Схема маршрутизации (URL‑структура).
- Разделение на клиентскую и серверную части.

#### 1.1. Пример схемы сайта
```markdown
/
├─ /about
├─ /products
│   ├─ /products/item1
│   └─ /products/item2
├─ /blog
│   ├─ /blog/post1
│   └─ /blog/post2
└─ /contact
```

### 2. Дизайн интерфейса (10 мин)
- Фон, цвета, типографика.
- Принципы визуальной иерархии.
- Навигационные элементы: меню, хлебные крошки, кнопки.

#### 2.1. CSS‑фреймворки (Bootstrap, Tailwind)
```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css">
```

### 3. Структура HTML‑документа (10 мин)
- Семантическая разметка: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`.
- Атрибуты `lang`, `meta` и их роль в SEO.

#### 3.1. Минимальный пример
```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Пример сайта</title>
</head>
<body>
  <header><h1>Мой сайт</h1></header>
  <nav><a href="/">Главная</a></nav>
  <main><p>Контент страницы.</p></main>
  <footer><p>&copy; 2025</p></footer>
</body>
</html>
```

### 4. Модульность и переиспользуемость (5 мин)
- Создание компонентов (HTML‑шаблоны, CSS‑модули).
- Файловая структура проекта.

#### 4.1. Пример структуры проекта
```
/src
  /components
    header.html
    footer.html
  /styles
    main.css
  index.html
```

### 5. Responsive Design (8 мин)
- Медиа‑запросы и гибкие сетки.
- Принцип mobile‑first.

#### 5.1. Медиа‑запрос
```css
@media (min-width: 768px) {
  .container {
    width: 750px;
  }
}
```

### 6. Кроссбраузерность и совместимость (5 мин)
- Применение префиксов, полифилов.
- Тестирование в основных браузерах.

## Выводы по лекции
- Архитектура сайта обеспечивает логическую иерархию и удобство навигации.
- Дизайн интерфейса повышает пользовательский опыт и влияет на SEO.
- Семантическая разметка улучшает доступность и индексацию.
- Модульность и responsive‑дизайн делают проект гибким и масштабируемым.
