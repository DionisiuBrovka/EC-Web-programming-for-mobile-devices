# Лабораторная работа № 13. Использование свойств и методов объекта window для организации взаимодействия с программой просмотра веб-страниц, определение местоположения пользователя

## Цель занятия
1. Научить студентов работать с объектом `window` в браузере для взаимодействия с пользователем.  
2. Познакомить студентов с API геолокации и показать, как получать данные о местоположении пользователя.

## Задачи занятия
- Освоить основные свойства и методы объекта `window` (alert, confirm, prompt, location, history).  
- Разработать скрипт, который открывает диалоговое окно, запрашивает данные у пользователя и изменяет URL страницы.  
- Изучить API `navigator.geolocation` и реализовать получение координат пользователя.  
- Проанализировать обработку ошибок при работе с геолокацией.  
- Провести практическую работу по созданию простого веб‑приложения, использующего полученные знания.

## План занятия (90 мин)

- Организационный момент — 5 мин  
- Актуализация знаний — 10 мин  
- Изучение нового материала — 20 мин  
- Практическая работа — 35 мин  
- Закрепление — 10 мин  
- Итог занятия — 10 мин  

## Содержание этапов

- **Организационный момент** – приветствие, проверка подключения к Интернету, раздача материалов, формулирование цели и задач.  
- **Актуализация знаний** – обсуждение предыдущих лабораторных работ, повторение работы с элементами DOM и событий.  
- **Изучение нового материала** – лекция о свойствах и методах `window`, демонстрация диалоговых окон, работы с URL, истории и геолокации.  
- **Практическая работа** – студенты пишут скрипт, который:  
  1. Запрашивает имя пользователя через `prompt`.  
  2. Выводит приветствие с помощью `alert`.  
  3. Меняет часть URL, используя `location`.  
  4. Запрашивает координаты через `navigator.geolocation.getCurrentPosition` и отображает их на странице.  
- **Закрепление** – групповой опрос, обсуждение ошибок, проверка результатов работы.  
- **Итог занятия** – подведение итогов, ответы на вопросы, раздача оценочного листа.

## Оборудование и материалы
- Компьютеры с доступом к Интернету.  
- Проектор и экран.  
- IDE (Visual Studio Code, PyCharm) или любой текстовый редактор.  
- Браузер Chrome/Firefox/Edge с включённой поддержкой geolocation.  
- Учебник/практическое руководство по JavaScript.  
- Сценарий лабораторной работы (PDF/Markdown).  

## Контроль и оценивание
- **Формативный контроль** – вопросы в ходе занятия, проверка кода в реальном времени.  
- **Итоговый контроль** – оценка выполненного кода по чеклисту: правильность вызовов `prompt/alert`, корректное изменение URL, успешное получение координат, обработка ошибок.  
- **Самостоятельная проверка** – студент заполняет чеклист и отправляет скриншоты результата.  

## Домашнее задание
1. Реализовать веб‑страницу, которая при каждом открытии запрашивает у пользователя разрешение на доступ к геолокации и выводит в консоль координаты.  
2. Добавить кнопку «Перейти» – при её нажатии страница должна перейти на URL, сформированный из введённого пользователем значения, с учётом текущего домена.
