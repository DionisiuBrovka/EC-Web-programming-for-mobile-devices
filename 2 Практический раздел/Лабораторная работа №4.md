# Лабораторная работа № 4. Создание и внедрение таблиц стилей с использованием различных видов селекторов и свойств в веб-документ различными способами. Кроссбраузерная верстка средствами CSS

## Цель и задачи

1. Изучить принципы работы таблиц стилей (CSS) и их влияние на структуру веб‑документа.
2. Научиться использовать различные типы селекторов (текстовые, атрибутные, псевдоклассы, псевдоэлементы) и свойства CSS.
3. Практически реализовать кроссбраузерную верстку с применением медиа‑запросов и префиксов.
4. Оценить совместимость и производительность полученного кода в разных браузерах.

## Теоретическая часть

| Понятие | Краткое определение |
|---------|---------------------|
| **CSS** | Язык таблиц стилей, описывающий внешний вид HTML‑документа. |
| **Селектор** | Элемент, который выбирает узлы DOM для применения к ним стилей. |
| **Селектор по типу** | `div`, `p` – выбирает элементы по тегу. |
| **Селектор по классу** | `.classname` – выбирает элементы с заданным классом. |
| **Селектор по ID** | `#idname` – выбирает элемент с конкретным ID. |
| **Атрибутный селектор** | `[attr="value"]` – выбирает элементы с указанным атрибутом. |
| **Псевдокласс** | `:hover`, `:active`, `:nth-child(n)` – добавляет стили при определённом состоянии. |
| **Псевдоэлемент** | `::before`, `::after` – вставляет контент до/после элемента. |
| **Медиа‑запрос** | `@media` – применяет стили в зависимости от параметров устройства. |
| **Браузерный префикс** | `-webkit-`, `-moz-`, `-ms-`, `-o-` – обеспечивает совместимость с разными движками. |

## Ход выполнения работы

1. **Создание структуры проекта**  
   - Создайте папку `lab4`.  
   - Внутри разместите файлы: `index.html`, `style.css`, `style.min.css`, `images/`.  
   ```html
   <!DOCTYPE html>
   <html lang="ru">
   <head>
     <meta charset="UTF-8">
     <meta name="viewport" content="width=device-width, initial-scale=1.0">
     <title>Lab 4 – CSS</title>
     <link rel="stylesheet" href="style.css">
   </head>
   <body>
     <!-- контент -->
   </body>
   </html>
   ```

2. **Определение базовых стилей**  
   - Установите шрифт, отступы, цвета в `body`.  
   ```css
   body {
     font-family: Arial, sans-serif;
     margin: 0;
     padding: 0;
     background-color: #f4f4f4;
     color: #333;
   }
   ```

3. **Стилизация заголовков и параграфов**  
   - Используйте селекторы по типу.  
   ```css
   h1, h2, h3 {
     margin-top: 1.5rem;
   }
   p {
     line-height: 1.6;
   }
   ```

4. **Создание сетки с помощью Flexbox**  
   - Добавьте класс `.grid` и примените `display: flex; flex-wrap: wrap;`.  
   ```css
   .grid {
     display: flex;
     flex-wrap: wrap;
     gap: 1rem;
   }
   .grid-item {
     flex: 1 1 200px;
     background: #fff;
     padding: 1rem;
     box-shadow: 0 2px 4px rgba(0,0,0,.1);
   }
   ```

5. **Использование атрибутных селекторов**  
   - Стилизация ссылок с определённым протоколом.  
   ```css
   a[href^="http://"] {
     color: #d00;
   }
   a[href^="https://"] {
     color: #06c;
   }
   ```

6. **Псевдоклассы и псевдоэлементы**  
   - Добавьте эффект наведения и стрелку к заголовку.  
   ```css
   h2:hover {
     color: #0066ff;
   }
   h2::after {
     content: " ➔";
     font-size: 0.8rem;
   }
   ```

7. **Медиа‑запросы для кроссбраузерной адаптивности**  
   - Сократите ширину колонок на устройствах ≤ 600px.  
   ```css
   @media (max-width: 600px) {
     .grid-item {
       flex-basis: 100%;
     }
   }
   ```

8. **Префиксы и совместимость**  
   - Добавьте префиксы для свойства `border-radius`.  
   ```css
   .rounded {
     -webkit-border-radius: 8px;
     -moz-border-radius: 8px;
     border-radius: 8px;
   }
   ```

9. **Минификация**  
   - Скопируйте содержимое `style.css` в `style.min.css` без пробелов и комментариев (можно использовать онлайн‑инструмент).  

10. **Тестирование в браузерах**  
    - Откройте `index.html` в Chrome, Firefox, Edge, Safari.  
    - Проверьте корректность отображения, отсутствие ошибок в консоли, совместимость медиа‑запросов.  

## Контрольные вопросы / задания для самопроверки

1. Что такое CSS‑селектор и какие типы селекторов существуют?  
2. Какой селектор выбирает элементы, у которых атрибут `data-role` равен `button`?  
3. В чем разница между `:hover` и `:active`?  
4. Как задать одинаковый отступ для всех сторон элемента через CSS?  
5. Как использовать медиа‑запрос для изменения фонового цвета при ширине экрана ≤ 480px?  
6. Какие префиксы нужны для свойства `transform: rotate(45deg);`?  
7. Что делает псевдоэлемент `::before` и как его можно использовать для добавления иконки?  
8. Как проверить кроссбраузерную совместимость таблицы стилей?  
9. Какой атрибут селектора выбирает ссылки, начинающиеся с `mailto:`?  
10. Как минифицировать CSS‑файл вручную и почему это важно?

## Отчёт по работе

Студент должен предоставить:

1. **Код проекта** – архив с файлами `index.html`, `style.css`, `style.min.css`, `images/`.  
2. **Пояснительный лист** – 1–2 страницы, где:
   - Описаны использованные типы селекторов и свойства.
   - Приведены примеры медиа‑запросов и префиксов.
   - Отмечены проблемы совместимости и способы их решения.
3. **Скриншоты** – минимум 3: в разных браузерах (Chrome, Firefox, Safari/Edge), на разных экранах (десктоп, планшет, мобильный).  
4. **Выводы** – кратко о том, какие навыки были получены, какие трудности возникли и как они решены.  

## Критерии оценки

1. Полнота и корректность кода (структура, отсутствие ошибок).  
2. Использование разнообразных селекторов и свойств.  
3. Кроссбраузерность: отсутствие ошибок в консоли и корректный вид во всех тестируемых браузерах.  
4. Точность и полнота отчёта (пояснения, скриншоты, выводы).  
5. Минификация и оптимизация (наличие файла `style.min.css`).
