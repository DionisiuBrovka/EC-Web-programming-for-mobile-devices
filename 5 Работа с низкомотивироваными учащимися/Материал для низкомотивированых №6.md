# Функции PHP для работы с БД MySQL

## Цель и задачи занятия
- Понять, как PHP взаимодействует с MySQL через функции и расширения.  
- Научиться подключать базу, выполнять запросы и обрабатывать результаты.  
- Практически применить знания на небольших проектах.  

## Почему это важно?
В реальной работе большинство веб‑сайтов хранит данные в БД. Без умения подключаться к MySQL и получать из него нужную информацию ваш сайт будет «пустой коробкой» – пользователь не получит ответов, а бизнес‑логика не сможет работать.

## Основные понятия и идеи
1. **mysqli** – расширение PHP для работы с MySQL (инструмент, который «говорит» с БД).  
2. **PDO** – объектно‑ориентированное API, более гибкое и безопасное, чем mysqli.  
3. **Коннект** – соединение с сервером БД.  
4. **Query** – запрос к БД (SELECT, INSERT, UPDATE, DELETE).  
5. **Result set** – набор строк, возвращаемый запросом.  
6. **Prepared statement** – подготовленный запрос, защищающий от SQL‑инъекций.  
7. **Fetch** – получение данных из результата (fetch_assoc, fetchAll).  

## Объяснение темы

### 1. Подключение к БД (≈10 мин)
```php
$mysqli = new mysqli("localhost", "user", "pass", "db");

if ($mysqli->connect_error) {
    die("Ошибка подключения: " . $mysqli->connect_error);
}
```
- *Пояснение*: аналог входа в банк – нужно имя, пароль и «коробка» (БД).  
- **Мини‑задача**: попробуй изменить имя пользователя и посмотри, что произойдёт.  

### 2. Выполнение простого SELECT (≈15 мин)
```php
$sql = "SELECT id, name FROM users";
$result = $mysqli->query($sql);

while ($row = $result->fetch_assoc()) {
    echo $row["id"] . " - " . $row["name"] . PHP_EOL;
}
```
- *Аналогия*: «список покупок» из таблицы.  
- **Попробуй сам!**  
  Что будет, если заменить `SELECT` на `SELECT * FROM users`?  

### 3. Добавление данных с помощью INSERT (≈10 мин)
```php
$name = "Alice";
$email = "alice@example.com";
$sql = "INSERT INTO users (name, email) VALUES ('$name', '$email')";
$mysqli->query($sql);
```
- *Игровая ситуация*: добавляем нового персонажа в игру.  
- **Вопрос**: как избежать ошибок при вставке одинаковых email?  

### 4. Безопасность: Prepared Statements (≈15 мин)
```php
$stmt = $mysqli->prepare("INSERT INTO users (name, email) VALUES (?, ?)");
$stmt->bind_param("ss", $name, $email);
$stmt->execute();
```
- *Пояснение*: как бы вы шифровали пароль, чтобы никто не смог его прочитать.  
- **Мини‑задача**: напиши prepared statement для обновления email по id.  

### 5. Работа с PDO (≈10 мин)
```php
$pdo = new PDO('mysql:host=localhost;dbname=db', 'user', 'pass');
$stmt = $pdo->query("SELECT * FROM users");
foreach ($stmt as $row) {
    echo $row['name'] . PHP_EOL;
}
```
- *Сравнение*: PDO – как универсальный переводчик между PHP и БД.  
- **Вопрос**: какие преимущества у PDO перед mysqli?
