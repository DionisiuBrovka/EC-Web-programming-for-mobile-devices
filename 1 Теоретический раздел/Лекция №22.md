# Получение и проверка данных формы средствами PHP

## Цели и задачи лекции
1. Понять механизм передачи данных из HTML‑формы в PHP‑скрипт.  
2. Научиться безопасно обрабатывать пользовательский ввод.  
3. Изучить методы валидации и фильтрации данных.  
4. Освоить основы защиты от распространённых угроз (XSS, CSRF, SQL‑инъекции).

## Ключевые понятия
- **GET** – метод отправки данных в URL‑строке.  
- **POST** – метод отправки данных в теле запроса.  
- **$_GET** – суперглобальный массив, содержащий параметры GET.  
- **$_POST** – суперглобальный массив, содержащий параметры POST.  
- **$_REQUEST** – объединённый массив GET, POST и COOKIE.  
- **filter_var()** – функция фильтрации переменных.  
- **htmlspecialchars()** – защита от XSS.  
- **prepared statements** – подготовленные запросы к БД.  
- **CSRF token** – уникальный токен для защиты от межсайтовой подделки запросов.  
- **isset() / empty()** – проверка существования и пустоты переменной.

## Основное содержание

### 1. Передача данных из формы (20 мин)
- Структура HTML‑формы: `<form method="post" action="process.php">`.  
- Различия между GET и POST.  
- Как PHP заполняет суперглобальные массивы.  
- Пример кода:

  ```html
  <form method="post" action="submit.php">
      <label>Имя: <input type="text" name="name"></label>
      <label>Почта: <input type="email" name="email"></label>
      <button type="submit">Отправить</button>
  </form>
  ```

  ```php
  // submit.php
  <?php
  $name = $_POST['name'] ?? '';
  $email = $_POST['email'] ?? '';
  echo "Привет, {$name}! Вы ввели e‑mail: {$email}";
  ?>
  ```

### 2. Проверка и фильтрация входных данных (25 мин)
- Проверка наличия (`isset()`) и непустоты (`!empty()`).  
- Фильтрация с `filter_var()` и `FILTER_VALIDATE_*`.  
- Пример:

  ```php
  $email = $_POST['email'] ?? '';
  if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
      die('Неверный формат e‑mail');
  }
  ```

- Удаление HTML-тегов: `strip_tags()`.  
- Экранирование для вывода: `htmlspecialchars()`.

### 3. Защита от XSS (10 мин)
- Что такое XSS и как он проявляется.  
- Использование `htmlspecialchars()` при выводе пользовательского контента.  
- Пример:

  ```php
  $comment = $_POST['comment'] ?? '';
  echo '<p>' . htmlspecialchars($comment, ENT_QUOTES, 'UTF-8') . '</p>';
  ```

### 4. Защита от SQL‑инъекций (15 мин)
- Почему обычный `mysqli_query()` опасен.  
- Использование подготовленных запросов (`mysqli_stmt_prepare`).  
- Пример:

  ```php
  $stmt = $mysqli->prepare('INSERT INTO users (name, email) VALUES (?, ?)');
  $stmt->bind_param('ss', $name, $email);
  $stmt->execute();
  ```

### 5. Защита от CSRF (10 мин)
- Суть проблемы.  
- Генерация токена: `$_SESSION['csrf_token'] = bin2hex(random_bytes(32));`.  
- Проверка токена на стороне сервера.  
- Пример кода:

  ```php
  // generate_token.php
  session_start();
  if (empty($_SESSION['csrf_token'])) {
      $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
  }
  ```

  ```html
  <form method="post" action="process.php">
      <input type="hidden" name="csrf_token" value="<?= $_SESSION['csrf_token'] ?>">
      <!-- остальные поля -->
  </form>
  ```

  ```php
  // process.php
  session_start();
  if (!hash_equals($_SESSION['csrf_token'], $_POST['csrf_token'] ?? '')) {
      die('CSRF token mismatch');
  }
  ```

## Примеры и иллюстрации
- Полный пример формы с валидацией и CSRF:

  ```html
  <!-- form.html -->
  <?php session_start(); ?>
  <?php
  if (empty($_SESSION['csrf_token'])) {
      $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
  }
  ?>
  <form method="post" action="process.php">
      <label>Имя: <input type="text" name="name" required></label><br>
      <label>Email: <input type="email" name="email" required></label><br>
      <label>Комментарий:<br>
      <textarea name="comment" rows="4" cols="50" required></textarea></label><br>
      <input type="hidden" name="csrf_token" value="<?= $_SESSION['csrf_token'] ?>">
      <button type="submit">Отправить</button>
  </form>
  ```

  ```php
  // process.php
  <?php
  session_start();
  // CSRF проверка
  if (!hash_equals($_SESSION['csrf_token'], $_POST['csrf_token'] ?? '')) {
      die('Ошибка CSRF');
  }
  // Валидация
  $name = trim($_POST['name'] ?? '');
  $email = trim($_POST['email'] ?? '');
  $comment = trim($_POST['comment'] ?? '');

  if ($name === '' || $email === '' || $comment === '') {
      die('Все поля обязательны');
  }

  if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
      die('Неверный e‑mail');
  }

  // Защита XSS
  $safe_comment = htmlspecialchars($comment, ENT_QUOTES, 'UTF-8');

  // Сохранение в БД (пример)
  $mysqli = new mysqli('localhost', 'user', 'pass', 'db');
  $stmt = $mysqli->prepare('INSERT INTO messages (name, email, comment) VALUES (?, ?, ?)');
  $stmt->bind_param('sss', $name, $email, $safe_comment);
  $stmt->execute();

  echo "Спасибо, {$name}! Ваш комментарий принят.";
  ?>
  ```

## Выводы по лекции
- Данные из форм передаются в PHP через суперглобальные массивы.  
- Безопасная обработка включает проверку, фильтрацию и экранирование.  
- Защита от XSS достигается `htmlspecialchars()`.  
- SQL‑инъекции предотвращаются при помощи подготовленных запросов.  
- CSRF‑атак можно избежать с помощью токенов, хранящихся в сессии.
