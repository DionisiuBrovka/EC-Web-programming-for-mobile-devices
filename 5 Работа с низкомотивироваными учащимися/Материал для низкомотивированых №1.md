# Создание пользовательских функций в JavaScript

## Цель и задачи занятия  
- Понять, что такое функция и зачем она нужна.  
- Научиться объявлять и вызывать собственные функции.  
- Освоить передачу аргументов и возврат значений.  
- Применить функции в простых практических задачах.

## Почему это важно?  
Функции – это строительные блоки любого программного продукта. Они позволяют разбивать сложные задачи на небольшие, легко управляемые части, упрощают повторное использование кода и делают программы понятнее. В реальной работе программист использует функции, чтобы ускорить разработку, уменьшить количество ошибок и облегчить сопровождение кода.

## Основные понятия и идеи  
- **Функция** – блок кода, который можно вызвать многократно.  
- **Объявление функции** – синтаксис, который описывает, как выглядит функция.  
- **Аргумент** – данные, которые передаются в функцию при вызове.  
- **Параметр** – переменная в объявлении функции, которая принимает значение аргумента.  
- **Возврат значения** – результат, который функция отдаёт наружу.  
- **Анонимная функция** – функция без имени, обычно используется как аргумент.  
- **IIFE (Immediately Invoked Function Expression)** – функция, которая вызывается сразу после объявления.

## Объяснение темы  

### [10 мин] Что такое функция?  
Функция похожа на кухонный прибор: ты ставишь туда ингредиенты (аргументы), включаешь прибор (вызываешь функцию), а он выдаёт готовое блюдо (результат).  

```js
function greet(name) {
  return `Привет, ${name}!`;
}
```

### [15 мин] Как объявить функцию?  
Используем ключевое слово `function`, имя функции, список параметров и тело в фигурных скобках.

```js
function add(a, b) {
  return a + b;
}
```

### [10 мин] Параметры и аргументы  
Параметры – это как ячейки в рецепте, а аргументы – конкретные ингредиенты. Если аргументы не переданы, в JavaScript они принимают значение `undefined`.

### [10 мин] Возврат значения  
Функция может возвращать данные. Если `return` не указан, функция возвращает `undefined`.

```js
function square(x) {
  return x * x;
}
```

> **Попробуй сам!**  
> Какой результат вернёт `square(3)`?

### [15 мин] Анонимные функции и IIFE  
Анонимные функции удобно использовать, когда нужно передать функцию как аргумент.

```js
setTimeout(function() {
  console.log('Через 1 секунду');
}, 1000);
```

IIFE – функция, которая выполняется сразу.

```js
(function() {
  console.log('IIFE выполнена');
})();
```

> **Попробуй сам!**  
> Напиши IIFE, которая выводит в консоль «Hello World».

### [10 мин] Практический пример: калькулятор  
Создадим набор простых функций для арифметики и используем их в главной функции.

```js
function add(a, b) { return a + b; }
function sub(a, b) { return a - b; }
function mul(a, b) { return a * b; }
function div(a, b) { return b !== 0 ? a / b : 'Ошибка: деление на ноль'; }

function calculator() {
  const x = 10;
  const y = 5;
  console.log('Сложение:', add(x, y));
  console.log('Вычитание:', sub(x, y));
  console.log('Умножение:', mul(x, y));
  console.log('Деление:', div(x, y));
}
calculator();
```

## Итог и выводы  
- Функции делают код более модульным и повторно используемым.  
- Параметры и аргументы позволяют передавать данные в функции.  
- Возврат значений делает функции полезными в дальнейших вычислениях.  
- Анонимные функции и IIFE расширяют гибкость кода.  
- Практика с простыми задачами закрепляет понимание.
