# Лабораторная работа № 18  
Проверка корректности информации, введенной пользователем в элементы формы. Обработка данных формы средствами JavaScript  

## Цель и задачи  
1. Научиться применять JavaScript для проверки и валидации пользовательского ввода в формах.  
2. Освоить работу с событиями формы (`submit`, `input`, `blur`) и механизмом отмены отправки.  
3. Научиться использовать регулярные выражения и условные конструкции для проверки разных типов данных.  
4. Разработать пользовательский интерфейс с визуальной обратной связью (цвет, сообщения об ошибках).  

## Теоретическая часть  
| Понятие | Описание |
|---------|----------|
| **DOM** | Объектная модель документа, позволяющая обращаться к элементам страницы. |
| **Event** | Событие, возникающее в результате действий пользователя (нажатие клавиши, ввод текста и т.п.). |
| **PreventDefault()** | Метод, отменяющий действие по умолчанию для события (например, отправку формы). |
| **Regular expression (RegExp)** | Набор символов, описывающих шаблон для поиска или проверки строки. |
| **Input type** | Атрибут `type` элемента `<input>` (text, email, number, etc.). |
| **Validation API** | Встроенный API браузера (`checkValidity()`, `setCustomValidity()`), упрощающий валидацию. |
| **CSS :valid / :invalid** | Селекторы для стилизации валидных и невалидных полей. |
| **FormData** | Объект, содержащий пары ключ‑значение всех полей формы, удобный для отправки на сервер. |
| **JSON.stringify()** | Преобразует объект JavaScript в строку JSON. |
| **AJAX (XMLHttpRequest/Fetch)** | Асинхронный запрос к серверу без перезагрузки страницы. |  

## Ход выполнения работы  

1. **Создайте HTML‑страницу с формой**  
   ```html
   <!DOCTYPE html>
   <html lang="ru">
   <head>
     <meta charset="UTF-8">
     <title>Валидация формы</title>
     <link rel="stylesheet" href="style.css">
   </head>
   <body>
     <form id="userForm" novalidate>
       <label>
         Имя:
         <input type="text" name="name" required minlength="2" maxlength="30">
         <span class="error"></span>
       </label>
       <label>
         Электронная почта:
         <input type="email" name="email" required>
         <span class="error"></span>
       </label>
       <label>
         Пароль:
         <input type="password" name="password" required minlength="6">
         <span class="error"></span>
       </label>
       <button type="submit">Отправить</button>
     </form>
     <script src="script.js"></script>
   </body>
   </html>
   ```
   *Пояснение*: `novalidate` отключает встроенную валидацию, чтобы использовать собственную.

2. **Добавьте стили для ошибок** (`style.css`)  
   ```css
   input:invalid { border-color: red; }
   .error { color: red; font-size: 0.9em; }
   ```
   *Пояснение*: визуальная подсказка при некорректном вводе.

3. **Напишите функцию проверки одного поля**  
   ```javascript
   function validateField(field) {
     const errorSpan = field.parentNode.querySelector('.error');
     if (!field.checkValidity()) {
       errorSpan.textContent = field.validationMessage;
       field.classList.add('invalid');
       return false;
     }
     errorSpan.textContent = '';
     field.classList.remove('invalid');
     return true;
   }
   ```
   *Пояснение*: использует `checkValidity()` и `validationMessage`.

4. **Обработайте событие `input` для каждой формы**  
   ```javascript
   const form = document.getElementById('userForm');
   form.addEventListener('input', (e) => {
     if (e.target.tagName === 'INPUT') validateField(e.target);
   });
   ```
   *Пояснение*: динамическая проверка при каждом вводе.

5. **Обработайте событие `submit`**  
   ```javascript
   form.addEventListener('submit', (e) => {
     e.preventDefault(); // отменяем отправку
     const fields = form.querySelectorAll('input');
     let allValid = true;
     fields.forEach(field => {
       if (!validateField(field)) allValid = false;
     });
     if (!allValid) return;

     const formData = new FormData(form);
     const data = Object.fromEntries(formData.entries());
     console.log('Отправляем данные:', JSON.stringify(data));

     // Пример отправки на сервер (можно закомментировать)
     // fetch('/api/submit', { method: 'POST', body: JSON.stringify(data), headers: { 'Content-Type': 'application/json' } })
     //   .then(r => r.json())
     //   .then(res => console.log(res));
   });
   ```
   *Пояснение*: сбор данных и логирование, отправка по желанию.

6. **Тестируйте работу**  
   - Введите корректные и некорректные данные.  
   - Проверьте, что ошибки исчезают при исправлении.  
   - Убедитесь, что форма не отправляется, пока есть ошибки.

## Критерии оценки  

1. **Корректность кода** – отсутствие синтаксических ошибок, правильная работа в современных браузерах.  
2. **Полнота проверки** – все поля проверяются, сообщения об ошибках понятны.  
3. **Качество оформления** – чистый HTML, читаемый CSS, понятные комментарии в JS.  
4. **Документирование** – отчёт содержит все требуемые разделы и демонстрацию результата.  
5. **Креативность** – дополнительные функции (например, подсказки, анимации) оцениваются по 0.5 балла.
