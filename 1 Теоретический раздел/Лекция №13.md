# Обработка данных формы средствами JavaScript

## Цели и задачи лекции  
1. Познакомить студентов с механизмом сбора и проверки данных из HTML‑форм.  
2. Выяснить роль событий и объектов `FormData`.  
3. Научить использовать JavaScript для валидации и отправки данных на сервер.  
4. Познакомить с практическими инструментами (Fetch API, async/await) для работы с формами.

## Ключевые понятия  
- **HTML‑форма** – элемент `<form>`, объединяющий поля ввода.  
- **Элемент ввода** – `<input>`, `<select>`, `<textarea>` и др.  
- **Событие `submit`** – триггер, срабатывающий при отправке формы.  
- **`FormData`** – объект, содержащий пары имя‑значение из формы.  
- **Валидация** – проверка корректности пользовательского ввода.  
- **Fetch API** – современный способ отправки запросов к серверу.  
- **async/await** – синтаксис для работы с асинхронным кодом.  
- **EventTarget** – интерфейс, реализуемый элементами, способными генерировать события.  
- **Кросс‑доменные запросы (CORS)** – механизм, позволяющий обращаться к ресурсам из другого домена.  
- **JSON** – формат обмена данными, часто используемый в веб‑приложениях.

## Основное содержание

### 1. Структура HTML‑формы (15 мин)  
- Разметка `<form>` и атрибуты (`action`, `method`, `enctype`).  
- Поля ввода: типы `text`, `email`, `number`, `password`, `checkbox`, `radio`, `file`.  
- Метки `<label>` и доступность.

```html
<form id="contactForm" action="/submit" method="POST">
  <label for="name">Имя:</label>
  <input type="text" id="name" name="name" required>

  <label for="email">E‑mail:</label>
  <input type="email" id="email" name="email" required>

  <button type="submit">Отправить</button>
</form>
```

### 2. Обработчик события `submit` (20 мин)  
- Предотвращение стандартного поведения (`event.preventDefault()`).  
- Получение доступа к объекту `FormData`.  
- Пример простого логирования значений.

```javascript
document.getElementById('contactForm').addEventListener('submit', function(e) {
  e.preventDefault(); // отменяем отправку браузером
  const formData = new FormData(e.target); // собираем данные
  console.log('Имя:', formData.get('name'));
  console.log('E‑mail:', formData.get('email'));
});
```

### 3. Валидация данных на клиенте (20 мин)  
- Встроенная валидация HTML5 (`required`, `pattern`, `min`, `max`).  
- Пользовательские проверки с помощью JavaScript.  
- Отображение сообщений об ошибках.

```javascript
function validateEmail(email) {
  const re = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return re.test(email);
}

document.getElementById('contactForm').addEventListener('submit', function(e) {
  e.preventDefault();
  const formData = new FormData(e.target);
  const email = formData.get('email');

  if (!validateEmail(email)) {
    alert('Введите корректный e‑mail');
    return;
  }
  // Отправка данных дальше
});
```

### 4. Отправка формы через Fetch API (25 мин)  
- Создание асинхронной функции `sendForm`.  
- Преобразование `FormData` в `URLSearchParams` или `JSON`.  
- Обработка ответа сервера и ошибок.

```javascript
async function sendForm(form) {
  const formData = new FormData(form);
  try {
    const response = await fetch(form.action, {
      method: form.method,
      body: formData,
      headers: { 'Accept': 'application/json' }
    });

    const result = await response.json();
    if (response.ok) {
      alert('Данные успешно отправлены');
    } else {
      alert('Ошибка сервера: ' + result.message);
    }
  } catch (err) {
    console.error(err);
    alert('Не удалось отправить данные');
  }
}

document.getElementById('contactForm').addEventListener('submit', function(e) {
  e.preventDefault();
  sendForm(e.target);
});
```

### 5. Работа с файлами и `enctype="multipart/form-data"` (10 мин)  
- Добавление поля `<input type="file">`.  
- Ограничения браузера при работе с файлами.  
- Пример отправки файла на сервер.

```html
<input type="file" name="avatar" id="avatar" accept="image/*">
```

```javascript
const form = document.getElementById('contactForm');
form.addEventListener('submit', async e => {
  e.preventDefault();
  const formData = new FormData(form);
  // FormData автоматически добавляет файл
  await sendForm(form);
});
```

## Примеры и иллюстрации  
- **Схема работы**  
  ```
  Пользователь → HTML‑форма → JS (submit) → валидация → Fetch → Сервер → Ответ
  ```

- **Таблица атрибутов `<form>`**  

| Атрибут | Значение | Описание |
|---------|----------|----------|
| `action` | URL | Адрес, куда отправляются данные |
| `method` | `GET`/`POST` | Способ передачи |
| `enctype` | `application/x-www-form-urlencoded`, `multipart/form-data` | Формат кодирования |

## Выводы по лекции  
- Формы — ключевой элемент взаимодействия пользователя с веб‑приложением.  
- JavaScript позволяет перехватывать, валидировать и отправлять данные без перезагрузки страницы.  
- `FormData` упрощает работу с полями ввода и файлами.  
- Асинхронные запросы через Fetch API делают взаимодействие с сервером гибким и эффективным.  
- Качественная валидация повышает надёжность и безопасность приложения.
