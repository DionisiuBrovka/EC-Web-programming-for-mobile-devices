# Функции PHP для работы с БД MySQL

## Цели и задачи лекции
1. Познакомить студентов с основами работы PHP с MySQL.  
2. Объяснить принципы подключения, выполнения запросов и обработки результатов.  
3. Научить использовать как расширение `mysqli`, так и объектно‑ориентированный интерфейс `PDO`.  
4. Разработать навыки безопасной работы с базой данных (протоколы защиты от SQL‑инъекций).  

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| MySQL | Система управления реляционными базами данных. |
| PHP | Серверный язык программирования, часто используется для веб‑приложений. |
| `mysqli` | Расширение PHP, обеспечивающее доступ к MySQL с процедурным и OO‑интерфейсом. |
| PDO | Объектно‑ориентированный слой доступа к БД, поддерживает несколько СУБД. |
| Prepared Statement | Подготовленный запрос, защищающий от SQL‑инъекций. |
| DSN | Data Source Name – строка соединения, указывающая тип БД и параметры доступа. |
| Transaction | Операционная единица, которую можно зафиксировать или откатить целиком. |
| Escaping | Экранирование специальных символов в строках. |
| Connection Pool | Пул соединений, позволяющий переиспользовать открытые подключения. |
| Error Handling | Обработка ошибок при работе с БД (try/catch, mysqli_error). |

## Основное содержание  

### 1. Подключение к MySQL (15 мин)  
- Подключение через `mysqli_connect`  
- Создание объекта `mysqli`  
- Подключение через PDO (`new PDO($dsn, $user, $pass)`)  
```php
// mysqli
$mysqli = new mysqli('localhost', 'user', 'pass', 'db');
if ($mysqli->connect_error) {
    die('Connect Error (' . $mysqli->connect_errno . ') '
        . $mysqli->connect_error);
}

// PDO
$dsn = 'mysql:host=localhost;dbname=db;charset=utf8mb4';
try {
    $pdo = new PDO($dsn, 'user', 'pass', [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION
    ]);
} catch (PDOException $e) {
    die('Connection failed: ' . $e->getMessage());
}
```

### 2. Выполнение запросов (25 мин)  
- Простой SELECT через `mysqli_query`  
- Выполнение INSERT/UPDATE/DELETE  
- Получение результатов (`fetch_assoc`, `fetchAll`)  
- Работа с подготовленными запросами  
```php
// Prepared statement (mysqli)
$stmt = $mysqli->prepare('SELECT * FROM users WHERE email = ?');
$stmt->bind_param('s', $email);
$stmt->execute();
$result = $stmt->get_result();
while ($row = $result->fetch_assoc()) {
    echo $row['name'];
}
$stmt->close();
```

```php
// Prepared statement (PDO)
$stmt = $pdo->prepare('SELECT * FROM users WHERE email = :email');
$stmt->execute(['email' => $email]);
foreach ($stmt as $row) {
    echo $row['name'];
}
```

### 3. Обработка ошибок и исключений (10 мин)  
- `mysqli_error` и `mysqli_errno`  
- Настройка `PDO::ATTR_ERRMODE`  
- Использование `try/catch` блоков  

### 4. Транзакции (10 мин)  
- Начало, фиксация и откат  
```php
$pdo->beginTransaction();
try {
    $pdo->exec("INSERT INTO accounts (balance) VALUES (100)");
    $pdo->commit();
} catch (Exception $e) {
    $pdo->rollBack();
    echo "Failed: " . $e->getMessage();
}
```

### 5. Безопасность (10 мин)  
- Защита от SQL‑инъекций через подготовленные запросы  
- Экранирование пользовательского ввода (`mysqli_real_escape_string`)  
- Ограничение прав пользователя в MySQL  

### 6. Практическое задание (10 мин)  
- Создание таблицы `students`  
- Заполнение данными  
- Реализация CRUD‑операций через `mysqli` и `PDO`  

## Примеры и иллюстрации  

```sql
-- Создание таблицы
CREATE TABLE students (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    age INT
);
```

```php
// Добавление студента (mysqli)
$stmt = $mysqli->prepare('INSERT INTO students (name, email, age) VALUES (?, ?, ?)');
$stmt->bind_param('ssi', $name, $email, $age);
$stmt->execute();
$stmt->close();
```

```php
// Получение всех студентов (PDO)
$stmt = $pdo->query('SELECT * FROM students ORDER BY id');
$students = $stmt->fetchAll(PDO::FETCH_ASSOC);
print_r($students);
```

## Выводы по лекции  
- PHP предоставляет два основных способа работы с MySQL: `mysqli` и `PDO`.  
- Подготовленные запросы являются ключом к безопасной работе с БД.  
- Транзакции позволяют гарантировать целостность данных.  
- Правильная обработка ошибок повышает надёжность приложений.
