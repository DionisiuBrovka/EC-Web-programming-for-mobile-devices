# Получение и проверка данных формы средствами PHP

## Цель занятия  
1. Научить студентов использовать суперглобальный массив `$_POST` для получения данных из HTML‑форм.  
2. Познакомить с техниками проверки и валидации пользовательского ввода в PHP.

## Задачи занятия  
- Понять структуру HTTP‑запросов и работу формы в браузере.  
- Научиться извлекать данные из `$_POST`.  
- Освоить методы проверки наличия и корректности введённых значений.  
- Создать простую форму с серверной обработкой и выводом результатов.  

## План занятия  

- Организационный момент — 5 мин  
- Актуализация знаний о HTTP и формах — 10 мин  
- Изучение нового материала: работа с `$_POST` — 25 мин  
- Практическое задание: проверка данных формы — 20 мин  
- Закрепление: обсуждение ошибок и лучших практик — 10 мин  
- Итог занятия и выводы — 10 мин  

## Содержание этапов  

- **Организационный момент**  
  Вступительное приветствие, проверка присутствия, краткое напоминание о предыдущем занятии.  

- **Актуализация знаний о HTTP и формах**  
  Обсуждение принципов работы GET/POST, роли атрибута `action`, разбор примеров простых форм.  

- **Изучение нового материала: работа с `$_POST`**  
  Демонстрация синтаксиса доступа к данным, вывод массива через `var_dump`, объяснение важности проверки существования ключей.  

- **Практическое задание: проверка данных формы**  
  Студенты реализуют форму регистрации с полями «имя», «email», «пароль» и серверной обработкой, включающей проверку заполнения, формат email и минимальной длины пароля.  

- **Закрепление: обсуждение ошибок и лучших практик**  
  Анализ типичных ошибок (отсутствие `isset`, прямой вывод данных), обсуждение способов защиты от XSS и CSRF.  

- **Итог занятия и выводы**  
  Краткое резюме ключевых понятий, раздача чек‑листов по валидации, объявление домашнего задания.  

## Оборудование и материалы  
- Компьютеры с доступом к интернету  
- Проектор и экран  
- IDE (PhpStorm, VS Code) или текстовый редактор  
- Серверный стек LAMP/LEMP (Apache/Nginx + PHP 8.x)  
- Учебник «PHP для начинающих» (разделы 5–6)  
- Пример кода на GitHub (репозиторий «php-form-validation»)

## Контроль и оценивание  
- **Краткие вопросы** (5 минут) по каждому этапу: что такое `$_POST`, зачем `isset`, как проверить email.  
- **Тест** из 5 вопросов с множественным выбором по теме валидации.  
- **Код‑ревью** выполнения практического задания: проверка наличия комментариев, корректности условий и выводов.  

## Домашнее задание  
1. Создать форму обратной связи с полями «имя», «email», «текст сообщения» и реализовать серверную проверку: все поля обязательны, email должен быть валидным, сообщение не более 500 символов.  
2. Добавить защиту от CSRF, используя токен в hidden‑поле формы.
