# Установка и настройка веб‑сервера. Основы синтаксиса PHP. Анализ ОКР №2  

Веб‑сервер и язык PHP — фундамент для разработки динамических веб‑приложений, API и микросервисов. Понимание их работы позволяет архитекторам проектировать масштабируемые решения и оптимизировать производительность, а также открывает доступ к современным инструментам DevOps и CI/CD.  

## Цели и задачи  

1. Освоить практические навыки установки и конфигурации Apache/Nginx с PHP‑FPM.  
2. Понять принцип работы PHP‑парсера, синтаксические особенности и их влияние на безопасность.  
3. Анализировать результаты ОКР №2: оценить эффективность выбранного стека, выявить узкие места и предложить пути улучшения.  
4. Развить аналитическое мышление через сравнение конфигураций и реальных кейсов.  
5. Способствовать исследовательскому подходу: формировать гипотезы, проводить эксперименты и фиксировать выводы.  

## Ключевые понятия  

| Термин | Расширенное определение | Зачем это нужно |
|--------|------------------------|-----------------|
| **Веб‑сервер** | Программа, обрабатывающая HTTP‑запросы и отдающая контент клиентам. | Базовый компонент веб‑инфраструктуры. |
| **Apache** | Открытый, модульный веб‑сервер, поддерживающий .htaccess и множество расширений. | Идеален для гибкой конфигурации и совместимости с legacy‑приложениями. |
| **Nginx** | Быстрый, событийно‑ориентированный сервер, часто используется как прокси‑сервер. | Высокая пропускная способность, низкие требования к памяти. |
| **PHP‑FPM** | FastCGI Process Manager, управляет процессами PHP для повышения производительности. | Улучшает масштабируемость и изоляцию процессов. |
| **php.ini** | Конфигурационный файл PHP, задающий параметры интерпретатора. | Позволяет тонко настроить безопасность и производительность. |
| **.htaccess** | Файл конфигурации Apache, применяемый к конкретному каталогу. | Удобен для быстрых правок без перезапуска сервера. |
| **SAPI** | Server Application Programming Interface – интерфейс взаимодействия PHP с веб‑сервером. | Позволяет выбирать оптимальный способ интеграции. |
| **OКР (Объектно‑концептуальный ресурс)** | Методы и структуры, используемые для моделирования бизнес‑логики. | Служит базой для анализа эффективности реализации. |
| **Код‑сниппет** | Минимальный фрагмент кода, демонстрирующий конкретную концепцию. | Быстрый способ проверить гипотезу. |
| **Benchmark** | Тестирование производительности, измеряющее время отклика и нагрузку. | Объективная оценка выбранного стека. |

## Примеры и иллюстрации  

### Конфигурация Nginx для PHP‑FPM  

```nginx
server {
    listen 80;
    server_name example.com;
    root /var/www/html;

    index index.php index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_pass unix:/var/run/php/php8.1-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }

    error_log /var/log/nginx/example_error.log;
    access_log /var/log/nginx/example_access.log;
}
```

### Мини‑скрипт PHP: вывод системной информации  

```php
<?php
header('Content-Type: application/json');

$info = [
    'PHP' => PHP_VERSION,
    'OS' => php_uname(),
    'SAPI' => PHP_SAPI,
    'Memory' => memory_get_usage(),
    'Time' => date('c')
];

echo json_encode($info, JSON_PRETTY_PRINT);
```

### Результаты benchmark (Apache vs Nginx)  

| Сервер | Avg. Response (ms) | Throughput (req/s) | CPU (%) |
|--------|--------------------|---------------------|---------|
| Apache | 125 | 240 | 42 |
| Nginx   | 68  | 410 | 27 |

> **Вывод**: при высоких нагрузках Nginx демонстрирует почти вдвое лучшую пропускную способность и более низкое потребление CPU.
