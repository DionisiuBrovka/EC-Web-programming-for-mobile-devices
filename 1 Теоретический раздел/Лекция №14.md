# Технология AJAX. Формат JSON и способы обработки данных в клиентских приложениях

## Цели и задачи лекции
- Познакомить студентов с основными принципами работы AJAX и его ролью в современных веб‑приложениях.
- Рассмотреть формат JSON как средство обмена данными между клиентом и сервером.
- Научить практическим способам отправки запросов и обработки ответов с помощью JavaScript.
- Развить навыки работы с асинхронным кодом и управления состоянием пользовательского интерфейса.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| **AJAX** | Асинхронный JavaScript и XML – технология, позволяющая обновлять части веб‑страницы без её перезагрузки. |
| **XMLHttpRequest** | Объект, обеспечивающий отправку HTTP‑запросов из браузера. |
| **Fetch API** | Современный интерфейс для выполнения сетевых запросов, заменяющий XMLHttpRequest. |
| **JSON** | JavaScript Object Notation – легковесный формат обмена данными, основанный на синтаксисе JavaScript‑объектов. |
| **CORS** | Cross‑Origin Resource Sharing – механизм, разрешающий доступ к ресурсам с другого домена. |
| **Callback** | Функция, передаваемая в другую функцию для выполнения после завершения операции. |
| **Promise** | Объект, представляющий результат асинхронной операции, который может быть выполнен, отклонён или в ожидании. |
| **async/await** | Синтаксический сахар над Promise, упрощающий написание асинхронного кода. |
| **REST** | Архитектурный стиль, использующий HTTP‑методы для взаимодействия с ресурсами. |
| **API‑эндпоинт** | URL‑адрес, через который клиент обращается к серверу. |

## Основное содержание (90 мин)

### 1. Введение в AJAX (15 мин)
- История и мотивация появления AJAX.
- Как AJAX изменил пользовательский опыт.
- Краткий обзор механизмов: `XMLHttpRequest` vs `Fetch API`.

```js
// Пример запроса через XMLHttpRequest
var xhr = new XMLHttpRequest();
xhr.open('GET', '/api/data');
xhr.onload = function() {
  if (xhr.status === 200) {
    console.log(JSON.parse(xhr.responseText));
  }
};
xhr.send();
```

### 2. Формат JSON (20 мин)
- Структура JSON: объекты, массивы, строки, числа, `true/false/null`.
- Сравнение JSON и XML: простота, читаемость, размер.
- Парсинг и сериализация в JavaScript (`JSON.parse`, `JSON.stringify`).

```js
// Сериализация объекта в JSON
var obj = {name: 'Alice', age: 30};
var json = JSON.stringify(obj);
// Парсинг JSON
var parsed = JSON.parse(json);
```

### 3. Работа с Fetch API (25 мин)
- Создание GET, POST, PUT, DELETE запросов.
- Обработка заголовков и тела запроса.
- Ошибки и исключения, `try/catch`.

```js
// GET запрос
fetch('/api/users')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));
```

### 4. Асинхронность: Promise, async/await (15 мин)
- Концепция Promise: `then`, `catch`, `finally`.
- Переход к async/await: упрощение синтаксиса, последовательность выполнения.
- Обработка ошибок в async/await.

```js
async function loadData() {
  try {
    const res = await fetch('/api/data');
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
```

### 5. Практика: простое CRUD‑приложение (15 мин)
- Структура проекта: `index.html`, `app.js`.
- Реализация: чтение, добавление, обновление, удаление пользователей через REST‑API.
- Обновление DOM без перезагрузки страницы.

```html
<ul id="user-list"></ul>
<script src="app.js"></script>
```

```js
// Пример обновления списка
async function refreshUsers() {
  const users = await fetch('/api/users').then(r => r.json());
  const list = document.getElementById('user-list');
  list.innerHTML = '';
  users.forEach(u => {
    const li = document.createElement('li');
    li.textContent = `${u.id}: ${u.name}`;
    list.appendChild(li);
  });
}
```

## Выводы по лекции
- AJAX позволяет обновлять части страницы без полной перезагрузки, улучшая UX.
- JSON – предпочтительный формат обмена данными благодаря простоте и компактности.
- Fetch API и async/await делают работу с сетью более удобной и читаемой.
- Понимание асинхронных паттернов критично для разработки современных клиентских приложений.
