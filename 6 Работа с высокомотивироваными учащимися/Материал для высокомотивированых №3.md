# Функции PHP для работы с БД MySQL

В современных веб‑приложениях взаимодействие с базой данных является центральной частью архитектуры. Знание того, как использовать PHP‑функции для подключения, выполнения запросов, обработки результатов и обеспечения безопасности, открывает двери к созданию масштабируемых, отказоустойчивых и безопасных систем. Для высокомотивированных студентов этот материал станет фундаментом, на котором можно строить собственные проекты, исследовательские работы и карьеру в области backend‑разработки.

## Цели и задачи

1. **Освоить базовые и продвинутые функции PHP для работы с MySQL** (mysqli, PDO).  
2. **Развить навыки проектирования безопасных запросов** (протоколирование, подготовленные выражения).  
3. **Понять принципы оптимизации и масштабирования** (пулинг соединений, кеширование результатов).  
4. **Сформировать критическое мышление** в выборе технологий, учитывая требования к производительности, безопасности и поддерживаемости.  
5. **Подготовить студентов к самостоятельному исследованию** и применению знаний в реальных задачах.

## Ключевые понятия

| Термин | Расширенное определение |
|--------|------------------------|
| **PDO (PHP Data Objects)** | Абстрактный слой доступа к БД, позволяющий использовать один и тот же API для различных СУБД, обеспечивая подготовленные выражения и транзакции. |
| **mysqli (MySQL Improved)** | Расширение PHP, предоставляющее объектно‑ориентированный и процедурный интерфейс к MySQL, поддерживает подготовленные выражения и асинхронные операции. |
| **Prepared Statement** | Преобработанный SQL‑запрос, где параметры передаются отдельно, что защищает от SQL‑инъекций и повышает производительность при повторном выполнении. |
| **Connection Pooling** | Хранение открытых соединений к БД в пуле для быстрого распределения среди запросов, уменьшая накладные расходы на установку соединения. |
| **Transaction** | Атомарный блок операций, который либо полностью завершается, либо откатывается, обеспечивая целостность данных. |
| **ORM (Object‑Relational Mapping)** | Технология, позволяющая работать с БД через объекты, скрывая детали SQL. |
| **Query Cache** | Кеширование результатов SELECT‑запросов, ускоряющий повторяющиеся запросы при неизменных данных. |
| **Stored Procedure** | Программный код, исполняемый на сервере БД, который может включать логику и возвращать результаты. |
| **ACID** | Свойства транзакций (Atomicity, Consistency, Isolation, Durability), гарантирующие надёжность операций. |
| **Indexing** | Создание индексов для ускорения поиска по столбцам, критически важный для масштабируемых приложений. |

## Примеры и иллюстрации

### Подключение через PDO

```php
<?php
$dsn = 'mysql:host=localhost;dbname=shop;charset=utf8mb4';
$options = [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    PDO::ATTR_PERSISTENT         => true, // Connection pooling
];
try {
    $pdo = new PDO($dsn, 'user', 'pass', $options);
} catch (PDOException $e) {
    die('Connection failed: ' . $e->getMessage());
}
?>
```

### Подготовленное выражение (mysqli)

```php
<?php
$mysqli = new mysqli('localhost', 'user', 'pass', 'shop');
$stmt = $mysqli->prepare('SELECT * FROM products WHERE category_id = ?');
$stmt->bind_param('i', $catId);
$stmt->execute();
$result = $stmt->get_result();
while ($row = $result->fetch_assoc()) {
    echo $row['name'];
}
$stmt->close();
$mysqli->close();
?>
```

### Транзакция

```php
<?php
$pdo->beginTransaction();
try {
    $pdo->exec("INSERT INTO orders (user_id, total) VALUES (1, 100)");
    $pdo->exec("INSERT INTO order_items (order_id, product_id, quantity) VALUES (LAST_INSERT_ID(), 5, 2)");
    $pdo->commit();
} catch (Exception $e) {
    $pdo->rollBack();
    echo "Failed: " . $e->getMessage();
}
?>
```
