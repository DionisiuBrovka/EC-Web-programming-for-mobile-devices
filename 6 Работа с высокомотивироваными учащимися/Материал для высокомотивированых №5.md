# Технология AJAX. Формат JSON и способы обработки данных в клиентских приложениях

В современной веб‑разработке взаимодействие клиент‑сервер происходит чаще всего асинхронно. AJAX‑запросы позволяют обновлять части страницы без перезагрузки, повышая отзывчивость интерфейсов и снижают нагрузку на сервер. Формат JSON стал стандартом обмена данными благодаря своей простоте, компактности и прямой совместимости с JavaScript. Понимание этих технологий открывает двери к созданию масштабируемых SPA, PWA и микросервисных архитектур, что критично для современных IT‑профессий.

## Цели и задачи

1. Овладеть механизмом асинхронных запросов в браузере: `XMLHttpRequest`, `fetch` и библиотеки (`axios`, `jQuery.ajax`).
2. Понять структуру и правила сериализации/десериализации данных в JSON.
3. Исследовать подходы к обработке JSON в клиентских приложениях: нативные методы, библиотеки (`jQuery`, `React`, `Vue`), паттерны (DTO, Repository).
4. Развить аналитическое мышление через сравнение AJAX‑технологий и альтернатив (WebSocket, Server‑Sent Events).
5. Подготовить основы для самостоятельного проектирования клиент‑серверного решения, используя AJAX и JSON.

## Ключевые понятия

| Термин | Расширенное определение |
|--------|--------------------------|
| **AJAX (Asynchronous JavaScript and XML)** | Техника обмена данными между клиентом и сервером без перезагрузки страницы. Позволяет улучшить UX, уменьшить трафик и повысить масштабируемость. |
| **JSON (JavaScript Object Notation)** | Текстовый формат, основанный на синтаксисе объектов JavaScript. Предоставляет лёгкую, читаемую структуру для передачи данных, совместимую с большинством языков. |
| **CORS (Cross‑Origin Resource Sharing)** | Политика безопасности, регулирующая доступ к ресурсам из других доменов. Необходима для корректного выполнения AJAX‑запросов в распределённых системах. |
| **REST (Representational State Transfer)** | Архитектурный стиль, использующий HTTP‑методы и URI для взаимодействия с ресурсами. Часто использует JSON как формат ответа. |
| **Data Transfer Object (DTO)** | Паттерн, описывающий структуру данных, передаваемых между слоями приложения, облегчая сериализацию и десериализацию. |
| **Repository Pattern** | Абстракция доступа к данным, позволяющая переключать источники (API, локальное хранилище) без изменения бизнес‑логики. |
| **WebSocket** | Протокол, обеспечивающий двунаправленную связь в реальном времени. Альтернатива AJAX при необходимости постоянного потока данных. |
| **Server‑Sent Events (SSE)** | Протокол, позволяющий серверу отправлять обновления клиенту через односторонний поток. Используется для простых push‑уведомлений. |
| **HTTP Cache** | Механизм сохранения ответов в клиенте, уменьшающий количество запросов и ускоряющий загрузку. |
| **Promise** | Объект, представляющий результат асинхронной операции. Важен для управления цепочками AJAX‑запросов. |

## Примеры и иллюстрации

```js
// Пример fetch‑запроса с обработкой JSON
fetch('https://api.example.com/users', {
  method: 'GET',
  headers: {
    'Accept': 'application/json'
  }
})
  .then(response => {
    if (!response.ok) throw new Error('Network error');
    return response.json();
  })
  .then(data => console.log('Users:', data))
  .catch(err => console.error(err));
```

```js
// Пример использования axios с DTO
import axios from 'axios';

class UserDTO {
  constructor({ id, name, email }) {
    this.id = id;
    this.name = name;
    this.email = email;
  }
}

axios.get('/api/users')
  .then(res => res.data.map(UserDTO))
  .then(users => {
    users.forEach(u => console.log(u.name));
  });
```

**Кейс из индустрии**:  
Spotify использует AJAX‑запросы для динамической загрузки плейлистов и рекомендаций, одновременно кэшируя ответы в IndexedDB. Это снижает время загрузки и позволяет работать offline.

**Схема** (обновление части страницы без перезагрузки):

```
Client  ──► AJAX (fetch) ──► Server
   ▲                                 │
   │                                 ▼
   └──────────── Response (JSON) ───┘
```
