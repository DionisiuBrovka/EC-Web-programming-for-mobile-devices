# Лабораторная работа № 6  
**Использование @media-запросов для создания адаптивной версии веб-страницы в зависимости от разрешения экрана и типа устройства**

## Цель и задачи
1. Познакомиться с концепцией адаптивного веб‑дизайна и ролью медиа‑запросов.  
2. Научиться прописывать правила CSS, применяемые при различных ширинах экрана и типах устройств.  
3. Реализовать простую адаптивную страницу и провести её тестирование на реальных устройствах и эмуляторах.  
4. Оценить влияние медиа‑запросов на производительность и удобство пользователя.

## Теоретическая часть
| Понятие | Описание |
|---------|----------|
| **Адаптивный веб‑дизайн** | Создание веб‑страниц, которые корректно отображаются на устройствах с разными экранами. |
| **Медиа‑запрос (`@media`)** | CSS‑директива, задающая стили, применяемые при выполнении условий (размер окна, ориентация и т.д.). |
| **Viewport** | Область, в которой отображается контент; задаётся через `<meta name="viewport">`. |
| **Breakpoint** | Пороговое значение ширины (или другого параметра), при котором меняется стиль. |
| **Классический подход** | Использование `max-width`/`min-width` в медиа‑запросах. |
| **Responsive Units** | Единицы измерения `vw`, `vh`, `em`, `rem` – динамически изменяемые, упрощающие адаптацию. |
| **Flexbox & Grid** | Современные модели раскладки, позволяющие легко менять порядок и размер элементов. |
| **Mobile‑first** | Стратегия проектирования сначала для мобильных устройств, затем расширение для десктопов. |
| **`orientation`** | Параметр медиа‑запроса, определяющий, как ориентировано устройство (portrait/landscape). |
| **`prefers-reduced-motion`** | Параметр, позволяющий отключать анимации для пользователей с нарушениями зрения. |

## Ход выполнения работы
1. **Подготовка проекта**  
   - Создайте папку проекта и два файла: `index.html` и `style.css`.  
   - В `index.html` добавьте базовую структуру и подключите CSS.  
   ```html
   <!DOCTYPE html>
   <html lang="ru">
   <head>
       <meta charset="UTF-8">
       <meta name="viewport" content="width=device-width, initial-scale=1.0">
       <link rel="stylesheet" href="style.css">
       <title>Адаптивная страница</title>
   </head>
   <body>
       <header>Header</header>
       <nav>Navigation</nav>
       <main>Content</main>
       <aside>Sidebar</aside>
       <footer>Footer</footer>
   </body>
   </html>
   ```

2. **Базовый стиль**  
   - В `style.css` задайте базовые стили для элементов (цвета, отступы).  
   ```css
   body { margin: 0; font-family: Arial, sans-serif; }
   header, nav, main, aside, footer { padding: 1rem; }
   nav, aside { background: #f0f0f0; }
   ```

3. **Определение breakpoints**  
   - Выберите 3–4 ключевых ширины: `576px`, `768px`, `992px`, `1200px`.  
   - Добавьте комментарии к каждому breakpoint.  

4. **Медиа‑запросы для мобильного первого подхода**  
   - Для ширины `<576px` используйте один‑колоночный макет.  
   - Для ширины `≥576px` добавьте вторую колонку (sidebar).  
   ```css
   @media (min-width: 576px) {
       main, aside { display: inline-block; vertical-align: top; }
       main { width: 70%; }
       aside { width: 28%; }
   }
   ```

5. **Адаптация для планшетов**  
   - При ширине `≥768px` измените порядок элементов (sidebar справа).  
   ```css
   @media (min-width: 768px) {
       nav { order: -1; }
   }
   ```

6. **Адаптация для десктопов**  
   - При ширине `≥992px` сделайте более широкие колонки и уменьшите отступы.  
   ```css
   @media (min-width: 992px) {
       main { width: 80%; }
       aside { width: 18%; }
   }
   ```

7. **Тестирование**  
   - Откройте страницу в браузере и измените размер окна.  
   - Используйте инструменты разработчика (Chrome DevTools → Toggle device toolbar) для эмуляции различных устройств.  
   - Проверьте, что элементы перестраиваются и стили применяются корректно.

8. **Оптимизация**  
   - Удалите неиспользуемые стили.  
   - Минимизируйте CSS (можно вручную).  

9. **Документирование**  
   - Включите в репозиторий README с описанием структуры проекта и инструкциями по запуску.  

## Контрольные вопросы / задания для самопроверки
1. Что такое медиазапрос и как он отличается от обычных CSS‑правил?  
2. Назовите три ключевых параметра, которые можно использовать в медиазапросах.  
3. Почему рекомендуется использовать mobile‑first подход?  
4. Как задать стили, которые применяются только при ширине экрана от 768 px до 991 px?  
5. Что делает `meta viewport` и почему он важен для адаптивного дизайна?  
6. Напишите медиазапрос, который активирует стили только при портретной ориентации устройства.  
7. Как можно уменьшить количество медиа‑запросов в проекте, не потеряв адаптивность?  
8. Опишите разницу между `min-width` и `max-width` в медиазапросах.  
9. Как проверить, что стили применяются корректно на разных устройствах без физического доступа к ним?  
10. Какие потенциальные проблемы могут возникнуть при использовании слишком большого количества медиа‑запросов?
